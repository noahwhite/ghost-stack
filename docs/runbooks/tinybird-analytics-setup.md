# TinyBird Analytics Setup Runbook

This runbook documents how to enable TinyBird analytics for the Ghost blog.

## Overview

TinyBird analytics provides real-time page view tracking and statistics for Ghost. The infrastructure supports zero-touch automated provisioning - you only need to add your TinyBird admin token once.

## Architecture

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     TINYBIRD ANALYTICS FLOW                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Browser → Caddy → /.ghost/analytics/* → traffic-analytics → TinyBird  │
│                                                                         │
│  Ghost Admin → Stats Dashboard → TinyBird API                           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## Prerequisites

1. TinyBird account at https://tinybird.co
2. TinyBird workspace created
3. Instance deployed with current infrastructure

## Initial Setup (One-Time)

### Step 1: Get TinyBird Admin Token

1. Log in to https://tinybird.co
2. Navigate to your workspace
3. Go to **Settings → Tokens**
4. Copy the **Admin Token** (starts with `p.eyJ...`)

### Step 2: Add Token to Instance

```bash
# SSH to instance
tailscale ssh core@ghost-dev-01

# Edit secrets file
sudo vim /var/mnt/storage/ghost-compose/.env.secrets

# Add these lines:
TINYBIRD_ADMIN_TOKEN=p.eyJ...your-admin-token...
TINYBIRD_API_URL=https://api.tinybird.co
```

For EU region, use:
```
TINYBIRD_API_URL=https://api.eu-central-1.aws.tinybird.co
```

### Step 3: Restart Services

```bash
# Restart provisioning and compose services
sudo systemctl restart tinybird-provision ghost-compose
```

The provisioning service will:
1. Sync TinyBird schema from Ghost
2. Deploy datasources and pipes to TinyBird (idempotent)
3. Extract the tracker token via `tb token ls`
4. Write tokens to `.env.generated`
5. Start Ghost with analytics enabled

## Verification

### Check Provisioning Status

```bash
# View provisioning logs
journalctl -u tinybird-provision

# Check generated tokens
cat /var/mnt/storage/ghost-compose/.env.generated
```

Expected output:
```
# Auto-generated by tinybird-provision.service - DO NOT EDIT
TINYBIRD_TRACKER_TOKEN=p.eyJ...
TINYBIRD_API_URL=https://api.tinybird.co
COMPOSE_PROFILES=analytics
```

### Check Services

```bash
# View running containers
docker compose -f /etc/ghost-compose/compose.yml ps

# Should show:
# - caddy
# - ghost
# - db
# - traffic-analytics (only if analytics profile is active)
```

### Check Traffic Analytics

```bash
# View traffic-analytics logs
docker logs $(docker ps -qf "name=traffic-analytics")
```

### Functional Test

1. Open your Ghost site in a browser
2. Visit a few pages
3. Go to Ghost Admin → Stats
4. Verify page views are being recorded (may take 30-60 seconds)

## Troubleshooting

### Provisioning Failed

Check the logs:
```bash
journalctl -u tinybird-provision -n 50
```

Common issues:
- **"TINYBIRD_ADMIN_TOKEN not found"**: Add token to `.env.secrets`
- **"Could not extract tracker token"**: Check TinyBird workspace/token permissions
- **"Tinybird source directory not found"**: Ghost image may not include TinyBird files

### Traffic Analytics Not Starting

Check if COMPOSE_PROFILES is set:
```bash
grep COMPOSE_PROFILES /var/mnt/storage/ghost-compose/.env.generated
```

Manually start with analytics profile:
```bash
cd /etc/ghost-compose
sudo docker compose --profile=analytics up -d
```

### Stats Not Showing in Ghost Admin

1. Check traffic-analytics is running
2. Verify Caddy is routing analytics requests:
   ```bash
   curl -I https://yourdomain.com/.ghost/analytics/health
   ```
3. Check TinyBird dashboard for ingested events

## Disabling Analytics

### Temporary (until next boot)

```bash
# Edit secrets file
sudo vim /var/mnt/storage/ghost-compose/.env.secrets
# Remove or comment out TINYBIRD_ADMIN_TOKEN

# Restart
sudo systemctl restart tinybird-provision ghost-compose
```

### Permanent

Remove TINYBIRD_ADMIN_TOKEN from `.env.secrets`. The provisioning service will skip setup and create an empty `.env.generated`, so Ghost will start without analytics.

## Environment Files

| File | Location | Purpose |
|------|----------|---------|
| `.env.config` | `/etc/ghost-compose/` | Non-secret config (via Ignition) |
| `.env.secrets` | `/var/mnt/storage/ghost-compose/` | Manual secrets (ADMIN_TOKEN) |
| `.env.generated` | `/var/mnt/storage/ghost-compose/` | Auto-generated (TRACKER_TOKEN) |

## Security Notes

- Admin token is stored in `.env.secrets` (0600 permissions)
- Tracker token is auto-generated to `.env.generated` (0600 permissions)
- No tokens are stored in Ignition/userdata
- Traffic analytics only exposes port 3000 internally
- All external traffic goes through Caddy with TLS
